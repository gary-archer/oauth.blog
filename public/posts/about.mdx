# About

I'm a long time [Software Engineer](https://www.linkedin.com/in/gary-archer-6b3613153/) with an interest in the art of separation. The deeper problems in software are often the result of insufficient design thinking, so that solutions do not align with goals and requirements. OAuth done right is a great example of solid design choices that provide high business value.

### Blog History

I wrote this blog's main content from 2017-2019, with a focus on non-functional coded requirements during modern API, web and mobile development. I still refine content occasionally, in line with design or technology improvements that enable my requirements to be met in more elegant ways.

### Blog Technology

The content is built as a basic NEXT.js blog. I can deploy the blog's static content to AWS CloudFront and ~400 global locations, to ensure that it performs well. The blog is an unsecured web application, whereas I prefer a pure SPA approach for secured web applications.
